

# Print Sales Order for Merchants

## Overview
Add a "Print Sales Order" button to the merchant order detail page that generates a printer-friendly PDF receipt. This helps sellers quickly print packing slips to include with shipments and streamline order processing.

## What the PDF Will Include
- Store name and logo at the top
- Order number and date
- Order status
- Buyer name, phone, email, and shipping address
- Order notes (if any)
- Itemized product list with quantities and prices
- Subtotal, shipping fee, and total amount
- Tracking number (if set)

Note: Platform fee will be excluded from the printed order since it's internal merchant info not meant for the buyer.

## Changes

### 1. Update MerchantOrderDetail component
- Add a "Print Order" button (using the Printer icon from lucide-react) in the header area next to the status badge
- Fetch the store name and logo URL from the `marketplace_stores` table (using the order's `store_id` -- this requires also fetching `store_id` from the order query)
- Implement a `printOrder()` function using `jsPDF` (already installed) that generates a clean, professional sales order PDF with all the details listed above
- The PDF will auto-download with filename like `order-{order_number}.pdf`

### Technical Details

**Data fetching update:**
- Add `store_id` to the order select query
- After fetching the order, query `marketplace_stores` for `store_name` and `logo_url` using the `store_id`

**PDF layout (using jsPDF):**
- Header: Store name (and logo if available), "SALES ORDER" title
- Order info: Order number, date, status
- Buyer section: Name, phone, email, address, notes
- Items table: Product name, qty, unit price, subtotal per row
- Totals section: Subtotal, shipping, total
- Tracking number if available
- Footer: "Generated by NoCap Pay" with timestamp

**Button placement:**
- Small icon button in the order detail header row, between the order info and the status badge
- Also works as a full-width button at the bottom of the page for easy access
